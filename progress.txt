## Context for AI Agents

### Project Structure
- Backend: `Tap2Dine/backend/` (FastAPI, Python)
  - `main.py` — all API endpoints
  - `database.py` — Database class with SQLite
  - `models.py` — Pydantic models
  - `ai.py` — Gemini AI recommendations
  - `.env` — environment variables (GEMINI_API_KEY, RAZORPAY keys)
- Frontend: `Tap2Dine/` (Vanilla HTML/CSS/JS)
  - `index.html` — customer menu page
  - `admin.html` — admin dashboard
  - `style.css` — main styles (dark theme)
  - `admin.css` — admin styles
  - `app.js` — customer JS
  - `admin.js` — admin JS

### Key Patterns
- Database: SQLite via `database.py`. All tables created in `init_db()`. Use `db.execute()` and `db.fetchall()` patterns.
- API: FastAPI. All endpoints in `main.py`. Models in `models.py`.
- Frontend: Vanilla JS fetch() to `http://localhost:8000/api/...`. Dark theme CSS.
- WebSocket: `manager.broadcast(event, data)` for real-time updates.

### Existing Tables
- `restaurants`, `menu_items`, `orders`, `reservations`, `bill_splits`

### Running the backend
- `cd Tap2Dine/backend && python -m uvicorn main:app --reload`

## Iteration Log

**Iteration 1: Analytics & Loyalty Feature Implementation**
- `backend/database.py`: Added `FeedbackORM`, `add_feedback`, `get_feedback`, `get_feedback_summary`, `get_revenue_stats`, `get_peak_hours`. (US-001, US-002)

- `admin.html` & `admin.js`: Upgraded Analytics tab. It now displays revenue cards (Today, Week, Month, All-Time), a link to the dedicated `analytics.html` page, and a new feedback summary section. Added `new_feedback` WebSocket listener to auto-refresh the UI without reloading. (US-007)
